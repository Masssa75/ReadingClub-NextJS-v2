üéØ The Three Learning Levels

Level 1: Listen + Hit ‚úÖ WORKING

Goal: Build letter-sound association through passive recognition

  • Letter falls from top of screen
  • System plays the Sound City Reading audio automatically
  • Student clicks/taps the letter when they see it
  • ‚úÖ Correct click ‚Üí Green explosion + score
  • ‚ùå Miss (letter reaches bottom) ‚Üí Lose 1 life

Learning Focus: Visual recognition + sound association (no production required)

Level 2: Listen + Choose ‚úÖ WORKING

Goal: Strengthen discrimination between similar sounds

  • 3 letters displayed on screen (multiple choice)
  • System plays ONE letter's sound
  • Student clicks the correct letter
  • ‚úÖ Correct choice ‚Üí Celebration + score
  • ‚ùå Wrong choice ‚Üí Try again (or lose life after 2 attempts)

Learning Focus: Auditory discrimination (telling sounds apart)

Level 3: Say + Hit ‚ö†Ô∏è NEEDS WORK

Goal: Independent sound production with voice recognition

  • Letter falls from top (NO audio plays)
  • Student must SAY the letter sound to make it explode
  • Voice recognition detects if sound matches (using calibrated patterns)
  • ‚úÖ Correct sound ‚Üí Explosion + score
  • ‚ùå Wrong/no sound ‚Üí Letter reaches bottom, lose life

Learning Focus: Independent production (saying sounds without prompts)

üîß Implementation Note: Requires calibrated voice patterns for current letter. If not calibrated ‚Üí prompt student to calibrate via modal.

üìö Batch Progression System

Philosophy: Like TypingClub, students can't skip ahead. They must master current content before unlocking new letters.

üé§ Calibration Integration

When Calibration is Needed

  • Level 1-2: No calibration required (system plays audio, student just clicks)
  • Level 3: Requires calibrated voice pattern for EACH letter
  • Before starting Level 3, game checks: "Is letter X calibrated?"
  • If NO ‚Üí Show modal: "Let's calibrate the sound for 'A' first!"

Calibration Flow

  1. Student reaches Level 3 for first time with letter 'A'
  2. Game detects: "A is not calibrated"
  3. Pause game, show message: "üé§ Before we play, let's teach the computer your voice!"
  4. Open calibration modal (existing proven system)
  5. Student completes 5 captures for 'A'
  6. Modal closes, save to Supabase
  7. Resume game at Level 3 - now voice recognition works!
‚úÖ Design Decision: Keep modal calibration SEPARATE from gameplay. Don't try to integrate calibration into falling letters (that was the failed experiment). Modal works great - use it!

Key Functions Needed

// Calibration check
function isLetterCalibrated(letter) {
    return calibrationData[letter] !== undefined;
}

// Prompt for calibration
function promptCalibration(letter) {
    pauseProgressGame();
    showMessage(`Let's calibrate '${letter}' first!`);
    openCalibrationModal(letter);
}

// Check mastery
function isLetterMastered(letter, level) {
    const progress = progressGame.letterProgress[letter];
    return progress[`level${level}`] >= 8;
}

// Check if batch complete at current level
function isBatchCompleteAtLevel(batchIndex, level) {
    const batchLetters = progressGame.batches[batchIndex];
    return batchLetters.every(letter =>
        isLetterMastered(letter, level)
    );
}

// Advance to next level
function advanceToNextLevel() {
    progressGame.currentLevel++;
    showCelebration("Level Complete!");
    saveProgressGameData();
}

2. Particle Effects: Custom Canvas Particles (~150 lines)

Why Custom Instead of Library:

  • ‚úÖ Zero dependencies (keeps single-file philosophy)
  • ‚úÖ Full control over appearance and behavior
  • ‚úÖ Tiny footprint (~0.1KB vs 5-20KB for libraries)
  • ‚úÖ Perfect for simple effects (explosions, confetti, sparkles)

Implementation:

class Particle {
    constructor(x, y, color) {
        this.x = x;
        this.y = y;
        this.vx = (Math.random() - 0.5) * 8;  // Random velocity
        this.vy = (Math.random() - 0.5) * 8;
        this.life = 1.0;                      // Fades out
        this.color = color;
        this.size = Math.random() * 4 + 2;
    }

    update() {
        this.x += this.vx;
        this.y += this.vy;
        this.vy += 0.3;        // Gravity effect
        this.life -= 0.02;     // Fade out
    }

    draw(ctx) {
        ctx.globalAlpha = this.life;
        ctx.fillStyle = this.color;
        ctx.fillRect(this.x, this.y, this.size, this.size);
    }
}

// Particle system manager
class ParticleSystem {
    constructor() {
        this.particles = [];
    }

    explode(x, y, color, count = 30) {
        for (let i = 0; i < count; i++) {
            this.particles.push(new Particle(x, y, color));
        }
    }

    update() {
        this.particles.forEach(p => p.update());
        this.particles = this.particles.filter(p => p.life > 0);
    }

    draw(ctx) {
        this.particles.forEach(p => p.draw(ctx));
        ctx.globalAlpha = 1.0; // Reset
    }
}

Why NOT Phaser.js or Pixi.js?

Decision: Keep Vanilla Canvas

  • ‚ùå Overkill: Phaser/Pixi adds 200-500KB+ for simple falling letters game
  • ‚ùå Complexity: Learning curve + refactoring existing working code
  • ‚ùå Bundle size: Breaks single HTML file architecture
  • ‚ùå Unnecessary features: Physics engines, tilemap systems, sprite managers not needed
  • ‚úÖ Current approach works: Vanilla Canvas is perfect for this simple game
  • ‚úÖ Performance: Canvas is fast enough for 3-6 year olds on tablets
  • ‚úÖ Maintainability: Less code = easier debugging

Frameworks are great for complex games with physics, collision detection, and many sprites. For falling letters + particle bursts, vanilla Canvas is ideal.

üéì Educational Philosophy

Why This Progression Works:

  • Level 1 (Passive): Builds neural connections between letter shapes and sounds without pressure
  • Level 2 (Guided): Strengthens discrimination - can they tell sounds apart?
  • Level 3 (Active): Independent production - the real test of mastery

Forced Sequential Mastery: Research shows distributed practice with mastery requirements is more effective than allowing students to skip ahead. Each level builds on the previous one.
